# utools-my-quick-bookmarks

一个放在 uTools 里的 Chrome 书签快捷入口。

它不是“浏览器里另一个复杂书签页”，而是把你最常用的书签查找、打开、回看和整理动作，尽量压缩到更短的路径里。

## 这个工具解决什么问题

很多人其实不是没有书签，而是书签越来越多以后，浏览器原生书签栏和书签管理页都不够顺手：

- 想找某个书签时，要先切到浏览器，再打开书签管理页，再慢慢翻。
- 书签很多时，只记得标题里的一个词、域名的一部分，或者记得它在某个目录里，但不记得完整名字。
- 一批经常打开的网站总会重复找、重复点，真正高频的入口没有被单独提出来。
- 当你只是想“立刻打开那个站”，浏览器书签管理器的层级和操作都偏重。

这个插件的目标，就是把这些动作收敛成一句话：

在 uTools 里呼出插件，直接搜，直接选，直接开。

## 为什么它更方便

相对“先打开浏览器，再打开书签管理器”的习惯，这个插件的便捷点主要在这几件事上：

- 入口更短：直接从 uTools 呼出，不用先切到 Chrome。
- 搜索更顺：直接使用 uTools 顶部输入框，不额外占页面空间，也更符合 uTools 的使用习惯。
- 匹配更准：支持搜索标题、域名和目录路径，找回“记得一点点”的书签更快，也能减少无关结果。
- 操作更轻：上下方向键切换目标，回车立即打开，整个过程几乎不用鼠标。
- 高频更突出：置顶、最近打开、打开次数会把真正常用的书签逐步收拢到前面。
- 状态可延续：主题、窗口高度、路径、置顶、最近打开等设置都会持久化到 uTools 数据库里，换设备同步后还能继续用。

## 适合谁用

如果你有下面这些习惯，这个插件会比较有价值：

- 书签很多，但经常重复访问其中一小部分站点。
- 平时已经把 uTools 当成日常入口，希望书签也走同一套“呼出即用”的方式。
- 更习惯键盘操作，不想频繁在浏览器里点层级。
- 会按目录整理书签，但真正找的时候常常只记得目录名或域名片段。

## 当前体验

目前这版已经可以完成一条完整的“快速找书签并打开”链路：

- 默认读取 `~/Library/Application Support/Google/Chrome/Default/Bookmarks`
- 首页以紧凑的 `Nothing` 风格单色卡片展示书签
- 直接使用 uTools 顶部输入框搜索 `标题 / 域名 / 目录路径`
- 多个关键词支持空格分词，并按 `AND` 逻辑同时匹配
- 支持方向键切换高亮卡片，按回车直接打开
- 点击卡片也可以用系统默认浏览器打开书签
- 支持插件内置顶 / 取消置顶
- 支持最近打开与打开次数
- 支持 `跟随系统 / 深色 / 浅色` 三态主题
- 支持调整插件窗口高度
- 设置页可以手填路径，也可以点击按钮直接选择本地 `Bookmarks` 文件
- 路径、主题、窗口高度、置顶、最近打开、打开次数都会持久化到 uTools 数据库
- 如果用户在另一台设备完成 uTools 数据同步，插件会重新读取这些设置

## 推荐怎么用

比较推荐的使用方式不是把它当成“完整书签后台”，而是把它当成你在 uTools 里的书签快速入口：

1. 用 uTools 的关键词呼出插件。
2. 直接在顶部输入框输入你记得的关键词。
3. 用上下方向键锁定目标，按回车打开。
4. 把最常打开的站点置顶，让首页越来越接近你的真实使用顺序。

如果你经常在不同设备之间切换，也建议把主题、窗口高度和书签路径先设好。只要 uTools 数据同步可用，大部分使用状态都能跟着过去。

## 当前限制

这版仍然是偏 MVP 的书签读取与打开工具，边界比较明确：

- 当前只支持 `macOS + Google Chrome`
- 默认路径只针对 Chrome 的 `Default` profile
- 如果你常用的不是这个 profile，需要在设置页改成对应的 `Bookmarks` 文件
- 同步过来的自定义书签路径如果在当前设备不可读，会临时回退到当前设备默认的 Chrome `Bookmarks` 路径
- 当前不会回写 Chrome 原书签文件
- 不支持在插件里修改书签名称、地址、位置，也不做重复书签检查

如果你要的是“快速找并打开书签”，这版已经够轻、够快；如果你要的是“完整管理浏览器书签”，那就不是它当前这轮的目标。
