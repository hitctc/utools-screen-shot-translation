# Nothing 风格样式收口设计

日期：2026-04-11
分支：`feature/custom-capture-pin`

## 1. 目标

当前页面虽然已经具备功能闭环，但整体视觉存在三个明显问题：

- 风格混杂，玻璃感、柔和阴影、厚重卡片和工业感排版同时存在
- 页面层级不够稳定，很多信息块之间会出现展示歧义
- 留白和 padding 不够克制，导致记录页和设置页看起来既不紧凑，也不够明确

本轮目标是把插件界面统一收口到更接近 Nothing 的视觉语言，但要结合当前中文桌面工具场景做本地化处理：

- 深色模式优先作为基线，浅色模式同步成套
- 所有核心文案保持中文，不做英文界面
- 强调工业感、信息面板感、明确边界和排版层级
- 主题强调色统一使用 `#d71921`
- 减少空白和视觉噪声，但保留必要的分区与呼吸感

## 2. 字体与加载

本轮使用的 Google Fonts：

- `Doto`
- `Space Grotesk`
- `Space Mono`

推荐加载方式：

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Doto:wght@400;500;600&family=Space+Grotesk:wght@400;500;700&family=Space+Mono:wght@400;700&display=swap" rel="stylesheet">
```

字体职责：

- `Space Grotesk`
  页面标题、正文、按钮、设置项、主要中文信息
- `Space Mono`
  编号、时间、状态标签、说明标签、次级元信息
- `Doto`
  只用于极少数品牌性或关键数字时刻，不进入正文和常规按钮

约束：

- 中文界面不强行使用 `Doto` 作为主标题字体
- 每个页面最多同时使用两种主要字体，`Doto` 只作为点缀

## 3. 视觉总原则

### 3.1 视觉语气

整体风格以 Nothing 的工业感为参考，但不直接复制其英文海报式大留白。

本项目应表现为：

- 黑白灰为主的技术面板
- 强排版层级而不是强特效
- 清晰边框而不是玻璃模糊
- 克制的圆角和轻量按钮
- 红色 `#d71921` 只作为强调和危险状态，不作为大面积装饰

### 3.2 不再使用的风格元素

本轮明确去掉或显著减弱：

- 玻璃拟态
- 大面积 backdrop blur
- 漂浮阴影作为主要层级手段
- 大块柔和渐变背景
- 过重的圆角大卡片

### 3.3 页面层级规则

每个页面只保留三层重要性：

- 第一层：页面标题 / 当前最核心状态
- 第二层：当前页面的主要内容区
- 第三层：编号、时间、标签、说明、辅助控制

目标是让用户第一眼知道：

- 这是哪一页
- 这里最重要的操作或信息是什么
- 次级信息在哪里，不会与主信息竞争

## 4. 颜色与主题

### 4.1 深色基线

深色模式作为第一优先基线：

- 页面底色：接近 `#000000`
- 一级表面：`#111111`
- 二级表面：`#1a1a1a`
- 主文字：接近 `#f2f2f2`
- 次级文字：接近 `#9a9a9a`
- 边框：接近 `#2a2a2a`
- 强调色：`#d71921`

深色感受应更接近“设备面板”而不是“磨砂玻璃”。

### 4.2 浅色同步

浅色模式不是纯白网页，而是偏暖白纸面：

- 页面底色：接近 `#f5f5f2`
- 一级表面：`#ffffff`
- 二级表面：接近 `#efefec`
- 主文字：接近 `#111111`
- 次级文字：接近 `#666666`
- 边框：接近 `#d6d6d2`
- 强调色：仍然是 `#d71921`

浅色模式应像技术手册，不像 SaaS 营销页。

## 5. 页面级设计

### 5.1 记录页

记录页改成“记录面板”结构，而不是“大标题 + 长说明 + 大块卡片”。

布局收口：

- 标题区高度收紧
- 标题说明保留一行短说明，不再拉成长段
- `主题状态 / 记录数量 / 列数控制` 合并成一条控制条
- 控制条是功能辅助，不应成为页面主视觉

记录列表规则：

- 保留瀑布流 / 多列浏览
- 缩略图区域必须足够高，保证图片可辨识
- 卡片信息区固定为：
  - 编号
  - 日期
  - 轻量动作按钮
- 叠层只显示必要内容，不再把顶部遮挡得太厚

记录页按钮：

- `重新钉住` 和 `删除` 做成轻按钮
- 右下角悬浮入口保留，但尺寸更轻，不喧宾夺主

### 5.2 设置页

设置页需要减少“卡片堆”的歧义，改成更明确的分区面板。

分组建议：

- 翻译配置
- 保存配置
- 界面配置

每组结构统一：

- 左上是中文分组标题
- 下方是一句短说明
- 控件本身排列清晰，不混排

设置页的“返回记录页”保留悬浮入口，但视觉权重低于分组内容。

### 5.3 结果页

结果页也要统一风格：

- 标题更明确
- 错误说明简洁
- 主要动作按钮不超过两个视觉重点
- 失败码作为次级标签处理，不要过度强调

## 6. 组件级规则

### 6.1 卡片

卡片是“信息容器”，不是装饰对象。

规则：

- 圆角控制在 `12px ~ 16px`
- 主要靠边框分区，不靠厚重阴影
- padding 统一，避免不同卡片像来自不同系统
- 卡片内容分段清楚，不要一块里混三种信息密度

### 6.2 缩略图

记录缩略图是记录页最关键的视觉内容。

规则：

- 优先保证可读性，不允许被压成细条
- 缩略图区域要有合理最小高度
- 裁切策略优先保证主体信息可见
- 编号与动作可叠在缩略图上，但信息层不要过厚

### 6.3 按钮

按钮整体收轻。

规则：

- 主要按钮才使用 `#d71921`
- 大多数按钮使用线框或轻底色
- 文字全部中文
- hover 只做轻微亮度 / 边框变化，不做夸张位移和发光

### 6.4 状态标签

状态标签统一小号胶囊，但弱化存在感。

规则：

- 普通状态走灰阶
- 错误、危险、关键提醒才使用 `#d71921`
- 时间、编号、状态统一用 `Space Mono`

### 6.5 滑块

列数滑块改成更仪表化的控制，而不是当前这种过于抢眼的横条。

规则：

- 左侧中文标签
- 右侧是当前值
- 轨道和手柄更克制
- 它是辅助控制项，不是主视觉中心

## 7. 间距与留白

本轮原则是“减少空白，但保留清晰分区”。

具体约束：

- 页面大区块之间的间距小于当前版本
- 卡片内部 padding 统一收紧
- 标题与说明之间保持紧凑关系
- 组件之间通过 `8 / 12 / 16 / 24` 的节奏控制，不再出现过大的断层

不做：

- 满屏空旷留白
- 靠巨大 margin 制造所谓高级感

## 8. 实现边界

预计会影响：

- `src/main.css`
- `src/screenTranslation/HomeView.vue`
- `src/screenTranslation/SettingsView.vue`
- `src/screenTranslation/ResultView.vue`
- 可能涉及 `src/App.vue` 的少量结构传递
- `README.md`
- `AGENTS.md`

如果字体加载要落到页面头部，可能还会涉及：

- `index.html`

## 9. 非目标

本轮不做：

- 新增页面
- 改动主流程逻辑
- 改动记录数据结构
- 重做自定义截图或钉住窗口交互
- 新增复杂动画
- 引入全新组件库

## 10. 验证

最小验证项：

1. 深色模式下记录页、设置页、结果页视觉统一
2. 浅色模式不是简单反色，而是独立可用
3. 所有中文文案不出现英文 UI 标签混杂
4. 记录卡片的缩略图区可读性提高
5. 按钮和状态标签更轻，不再抢占主层级
6. 页面空白减少，但内容分区仍清晰
