# jQuery 面板迁移设计稿

## 1. 目标

把当前 Vue 承载的 `钉住记录 / 设置 / 失败结果` 三个页面，完整迁移到新的静态 `HTML + jQuery + preload` 运行壳里。

这次迁移的重点不是继续改主流程，而是保证：

- `截屏翻译钉住` 继续保持无页面直达
- `钉住记录 / 设置 / 失败结果` 在 jQuery 壳里具备和原 Vue 版等价的功能
- 样式和视觉结果尽量延续当前 Vue 版
- 实现结构比原 Vue 版更简单，不重新造一套 jQuery 组件系统

## 2. 范围

本轮迁移只覆盖：

- `钉住记录`
- `设置`
- `失败结果`

本轮不做：

- 重写截图、翻译、钉图主流程
- 改百度 V2 业务边界
- 改 pin window 交互
- 改保存目录和总清单格式
- 重做主题系统
- 删除旧 `src/` 与 `vite.config.js`

## 3. 迁移策略

采用“行为等价，结构简化”的策略。

含义是：

- 用户看到和用到的页面结果，尽量与当前 Vue 版保持一致
- 但 jQuery 实现不追求模板、组件和 DOM 结构逐节点 1:1 复刻
- 页面功能、状态和关键交互必须完整带过来
- 内部实现优先使用少量渲染函数和集中事件绑定，避免再次长成一坨巨型脚本

## 4. 新前端壳结构

新的静态前端壳只保留下面这些文件：

- `public/index.html`
  - 页面根节点
  - records/settings/result 三个容器
  - 引入 jQuery 与运行脚本
- `public/app.js`
  - 启动入口
  - 面板初始化事件接收
  - 页面级状态协调
  - 全局事件绑定
- `public/app.css`
  - 继续承接当前 Vue 版主要样式 token 和核心类名
- `public/panel-state.js`
  - 最小状态模型
  - 当前视图、结果页状态、records/settings 数据快照
- `public/panel-renderers.js`
  - records/settings/result 的 HTML 渲染函数

这 5 个文件的边界是固定的：

- `app.js` 不直接拼大段 HTML
- `panel-renderers.js` 不做系统调用
- `panel-state.js` 不做 DOM 操作
- `app.css` 只承接样式，不带业务状态逻辑

## 5. records 页迁移要求

需要从 Vue 版完整迁过来的能力：

- 读取保存目录总清单
- 空态展示
- warning 展示
- 瀑布流记录卡片
- 默认 3 列，支持滑块调到 3~6 列并持久化
- 点击缩略图或按钮重新钉住
- 删除记录
- 删除时遵守 `confirmBeforeDelete`
- 右下角设置按钮
- 当前主题状态展示

实现上允许简化：

- 卡片 DOM 不要求和 Vue 版完全一致
- 但必须保留：
  - 缩略图
  - 编号
  - 时间
  - 重新钉住
  - 删除

## 6. settings 页迁移要求

需要从 Vue 版完整迁过来的能力：

- 百度凭证输入
  - `AppID`
  - `Access Token`
- 当前凭证状态条
- 百度文档跳转
- 百度开放平台跳转
- 翻译方向选择
- 当前翻译方向状态条
- 保存结果开关
- 保存目录输入
- 选择目录
- 打开目录
- 保存目录 warning
- 删除前确认开关
- 当前删除行为状态条
- 主题模式选择
- 当前主题状态条
- 窗口高度滑块
- 恢复默认高度
- 返回记录页按钮

实现上允许简化：

- 不要求继续使用 Vue 的 `computed`
- 但所有设置项的持久化语义必须保持不变

## 7. result 页迁移要求

需要迁过来的能力：

- 失败码映射
- 标题、说明、下一步提示
- 重试
- 前往设置
- 返回记录页
- 当前主题状态展示

结果页仍然只承担失败态，不承担成功流程页面。

## 8. 样式迁移原则

这次不重新设计样式。

样式迁移原则是：

- 直接复用当前 Vue 版已经验证过的主类名、token 和层级
- 视觉结果尽量与 Vue 版一致
- 不重新发明一套 jQuery 专用命名体系
- 不为“更纯粹”而大规模重写 CSS
- 如果某些类只是 Vue 结构用的中间层，而 jQuery 不再需要，可以删，但不能影响视觉结果

## 9. 实现顺序

严格按下面顺序迁移：

1. records 页
2. settings 页
3. result 页
4. 清理当前 `public/app.js` 里的临时占位逻辑

这样做是为了先把最重的 records/settings 承载面迁完，再把结果页收尾。

## 10. 验证要求

迁移完成后，至少验证：

- `钉住记录`
  - 能正常进入
  - 列数滑块生效
  - 删除逻辑生效
  - 重新钉住可用
- `设置`
  - 所有输入项和开关都可读写
  - 目录按钮可用
  - 外链跳转可用
  - 返回记录页可用
- `失败结果`
  - 失败码映射和按钮动作可用

此外必须跑：

- 最相关前端测试
- `npm run build`

## 11. 非目标

这次迁移不解决下面这些问题：

- 开发态白框问题
- pin window 视觉问题
- 百度 V2 块级回填效果优化
- 主流程再次重写

这些都不属于本次“把 Vue 承载面完整迁到 jQuery 壳”的范围。
